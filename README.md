# 🚗 CarSpotter MCP Server

An MCP (Model Context Protocol) server that lets you add cars to your CarSpotter quiz game directly from Claude. Snap a photo, send it to Claude, and it's in the game.

## What It Does

Three tools available to Claude:

- **add_car** — Send a car photo, Claude analyzes it (make, model, era, hints, distractors) and uploads to your game database
- **list_cars** — See all cars currently in your game
- **remove_car** — Delete a car by make/model

## Setup

### 1. Clone and Install

```bash
git clone https://github.com/YOUR_USERNAME/carspotter-mcp.git
cd carspotter-mcp
npm install
```

### 2. Create `.env` File

Copy the example and fill in your keys:

```bash
cp .env.example .env
```

Edit `.env`:

```
ANTHROPIC_API_KEY=sk-ant-api03-your-key-here
SUPABASE_URL=https://xqjmqwgkrfvofukuxwhb.supabase.co
SUPABASE_SERVICE_KEY=your-supabase-service-role-key
PORT=3000
```

**Where to find these:**
- `ANTHROPIC_API_KEY`: console.anthropic.com → API Keys
- `SUPABASE_URL`: Your Supabase project URL (you already have this in Vercel)
- `SUPABASE_SERVICE_KEY`: Supabase → Settings → API → `service_role` key (the secret one, NOT the anon key)

### 3. Build and Run Locally

```bash
npm run build
npm start
```

You should see:
```
🚗 CarSpotter MCP Server running on port 3000
   MCP endpoint: http://localhost:3000/mcp
   Health check: http://localhost:3000/
```

### 4. Test It

Visit http://localhost:3000 — you should see a JSON health check response.

## Deploy to Railway (Free Tier)

Railway gives you a public URL so Claude.ai can reach your server from anywhere (including mobile).

### Step 1: Push to GitHub

Create a new repo on GitHub called `carspotter-mcp`, then:

```bash
cd carspotter-mcp
git init
git add .
git commit -m "CarSpotter MCP server"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/carspotter-mcp.git
git push -u origin main
```

### Step 2: Create Railway Project

1. Go to [railway.app](https://railway.app) and sign in with GitHub
2. Click **"New Project"** → **"Deploy from GitHub Repo"**
3. Select your `carspotter-mcp` repo
4. Railway will auto-detect it as a Node.js project

### Step 3: Add Environment Variables

In Railway, go to your project → **Variables** tab and add:

```
ANTHROPIC_API_KEY=sk-ant-api03-your-key-here
SUPABASE_URL=https://xqjmqwgkrfvofukuxwhb.supabase.co
SUPABASE_SERVICE_KEY=your-supabase-service-role-key
PORT=3000
```

### Step 4: Configure Build

In Railway → **Settings** tab:
- **Build Command**: `npm run build`
- **Start Command**: `npm start`

Railway should auto-deploy. Once done, you'll get a public URL like:
```
https://carspotter-mcp-production.up.railway.app
```

### Step 5: Verify

Visit your Railway URL — you should see the health check JSON.

## Connect to Claude

### Claude.ai (Web & Mobile)

1. Go to **claude.ai** → click your profile → **Settings**
2. Click **Connectors** in the sidebar
3. Scroll down and click **"Add custom connector"**
4. Enter your MCP server URL: `https://YOUR-RAILWAY-URL.up.railway.app/mcp`
5. Click **Add**

That's it! Claude now has access to your CarSpotter tools.

### Claude Desktop (Optional)

Edit your Claude Desktop config file:

**Mac**: `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "carspotter": {
      "command": "npx",
      "args": [
        "mcp-remote@latest",
        "https://YOUR-RAILWAY-URL.up.railway.app/mcp"
      ]
    }
  }
}
```

Restart Claude Desktop.
